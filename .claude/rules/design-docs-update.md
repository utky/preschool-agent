# ルール: plan mode後の設計書更新（強制）

## 適用タイミング
plan modeでの設計が承認され、実装を開始する前。

## 必須アクション

実装コードを1行も書く前に、以下を完了すること：

1. **`docs/design/10_slice_plan.md`** を更新する
   - 該当スライスの設計詳細を実際の実装方針に合わせて修正
   - ファイル名・モデル名・API名などが変わっている場合は反映する

2. **関連する設計書（`docs/design/`配下）** を必要に応じて更新する
   - アーキテクチャ変更 → `01_architecture.md`
   - API変更 → `06_api_design.md`
   - フロントエンド変更 → `07_frontend_design.md`
   - インフラ変更 → `08_infrastructure.md`

3. 設計書の変更を含む**コミットを作成**してから実装を開始する

## 禁止事項

- 設計書を更新せずに実装ファイルを編集・作成すること
- 設計書更新コミットを実装コミットと混ぜること
